# Micro ECF Troubleshooting

Start with:

```bash
micro-ecf doctor --dir .
micro-ecf lint ECF.md
```

Inspect the JSON `ok`, `checks`, `findings`, and `next_steps` fields. Some readiness failures are structured results and do not automatically produce a nonzero process exit code.

## `install` Says Approval Is Required

This is the expected fail-closed flow. Review the plan first:

```bash
micro-ecf plan --dir .
```

If you approve the listed local writes, run:

```bash
micro-ecf install --dir . --yes
```

Do not add `--yes` to unattended automation unless the owner has already approved that exact write scope.

## `doctor` Reports Missing Artifacts

`doctor` expects `ECF.md`, `AGENTS.md`, `MICRO_ECF_LLM_BOOTSTRAP.md`, and the normal `.micro-ecf` policy, source map, context packet, policy summary, deployment preview, and Harness export.

Run `plan`, review it, then run the approved install. If you use a non-default output directory, pass the same `--output-dir` to `doctor`.

## `lint` Reports Missing Front Matter Or Tokens

`ECF.md` must start with YAML front matter delimited by `---`. Required tokens are `version`, `project`, `scope`, `allowed_sources`, and `blocked_sources`. A missing `agent_os` token or a missing recommended Markdown section is a warning; missing core tokens are errors.

Compare a known-good starter generated by `micro-ecf init` and rerun:

```bash
micro-ecf lint ECF.md
```

## A File Appears Under `blocked`

Check its recorded reason in `.micro-ecf/source-map.json`:

- a policy or secret-like path encountered during traversal is intentionally unavailable;
- `unsupported_or_binary_file` means the file type is not part of the text/SQLite source surface;
- `max_file_bytes_exceeded` means it exceeded the default 200,000-byte limit or your override;
- `max_files_limit_reached` means the included-source limit, 500 by default, was reached;
- generated Micro ECF artifacts are excluded to the separate `generated` array to avoid self-indexing;
- default-skipped directories such as `.git`, `node_modules`, and `.micro-ecf` may not appear in either preview because the walker never enters them.

Adjust explicit size/count limits only after reviewing the additional context and secret-exposure risk. Do not loosen blocked-source rules merely to make a count green.

## `validate-policy` Fails

Confirm the path passed with `--policy` exists and contains valid JSON matching the expected Micro ECF policy shape. By default the CLI reads `.micro-ecf/policy.json` relative to the current directory.

## `index` Says A Path Does Not Exist

The positional path is resolved locally and must exist before traversal. Correct the path and rerun; the thrown error is printed as JSON on stderr and exits `1`.

## `export` Says Only `--agent-os` Is Supported

Micro ECF V1 requires the explicit export mode:

```bash
micro-ecf export --agent-os
```

This writes a local Harness handoff only. It does not authorize a hosted action.

## `search` Cannot Find A Source

`search` reads `source-map.json` under `--root` (default `.micro-ecf`) and performs substring matching over path, type, and summary; it is not semantic retrieval. Re-run `index` after source changes, verify the file is in `sources` rather than `blocked` or `generated`, and check the requested query and `--limit`.

## Worklog Checkpoint Or Finish Says There Is No Active Worklog

Start one first with `micro-ecf worklog begin --goal "..."`. Checkpoint and finish operations intentionally refuse to invent session state.

## MCP Configuration Is Present But Tools Do Not Appear

Preview the generated configuration before installing it:

```bash
micro-ecf mcp-config --target codex --dir .
```

Use `--write` or `--install-codex` only for the intended workspace/Codex home, then restart the MCP client. The stdio server must keep stdout reserved for JSON protocol messages.

## A Required Context Provider Is Unavailable

Micro ECF does not silently replace an unavailable provider with hidden RAG. Review the provider contract, `required` and `required_for_action_classes` settings, then restore the declared provider or keep the affected action fail-closed. See [Provider Wrapping](../PROVIDER_WRAPPING.md).

## The Harness Export Exists But Agent OS Is Not Deployed

That is expected. `micro-ecf export --agent-os` only writes a local handoff. A separate owner-reviewed Agent OS readiness/preview flow is required, and provisioning, wallet funding, public exposure, selling, and x402 remain separately gated. See [Micro ECF To Agent OS](../MICRO_ECF_TO_AGENT_OS.md).

## The CLI Exited `0` But `ok` Is `false`

Structured refusals and some readiness reports are encoded in JSON. Inspect `ok`, `requires_approval`, checks, and findings in automation. See the [CLI exit behavior](./CLI_REFERENCE.md#exit-behavior).
